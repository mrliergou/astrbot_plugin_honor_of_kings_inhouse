# 王者荣耀内战报名分组插件

AstrBot 插件，用于王者荣耀内战的报名和智能分组。

## 功能特性

### 核心功能
- ✅ **报名系统**：支持随机模式和智能模式报名
- ✅ **随机分组**：10人随机分配到两队
- ✅ **智能分组**：根据玩家擅长分路智能分配
- ✅ **数据持久化**：报名数据自动保存，支持重启恢复
- ✅ **历史统计**：对局记录查询和胜率统计
- ✅ **管理员权限**：管理员可清空报名池

### 进阶特性
- 🎯 **约束优化算法**：MCMF 算法保证位置覆盖
- ⏰ **数据过期机制**：5小时自动过期 + 午夜重置
- 📊 **结构化看板**：Emoji 语义化展示
- 🔒 **并发控制**：群组级锁保证数据一致性

## 安装

1. 将插件目录复制到 AstrBot 的插件目录
2. 重启 AstrBot

## 使用方法

### 报名相关

```
/报名                    # 随机分组模式报名
/报名 中单 打野          # 智能分组模式报名（指定擅长分路）
/报名 中 野              # 支持缩写：上/野/中/射/辅
/取消报名                # 取消当前报名
/报名列表                # 查看当前报名情况
```

### 分组相关

```
/随机分组                # 10人随机分配
/智能分组                # 根据擅长分路智能分配
```

### 历史统计

```
/历史对局                # 查看最近10场对局
/胜率统计                # 查看红蓝方胜率
/归档对局 红             # 归档对局结果（红/蓝/平）
```

### 管理员命令

```
/清空报名                # 清空当前报名池（需要管理员权限）
```

| 分路 | 缩写 | Emoji |

| 打野 | 野/刺 | ⚔️ |

| 射手 | 射/后 | 🏹 |
| 辅助 | 辅/肉/盾 | 🚀 |

## 智能分组算法

插件提供两种智能分组算法：

1. **贪心+回溯算法**（greedy）
   - 优先锁定唯一位置的用户
   - 贪心分配多选用户
   - 1-2步回溯优化

2. **最小费用最大流算法**（mcmf，默认）
   - 多次随机采样
   - 评分选优
   - 保证位置覆盖和偏好匹配

## 配置

插件会自动为每个群组创建配置，默认配置如下：

```json
{
  "ttl_hours": 5,
  "max_history": 500,
  "admin_users": [],
  "algorithm": "mcmf"
}
```

## 数据存储

数据存储在 `data/hok_inhouse_state.json`，包含：
- 各群组的报名池
- 历史对局记录
- 群组配置

## 技术架构

- **分层设计**：Presentation → Application → Domain → Infrastructure
- **并发控制**：asyncio.Lock 群组级锁
- **原子写入**：临时文件 + 原子替换
- **备份恢复**：自动备份 + 损坏恢复
