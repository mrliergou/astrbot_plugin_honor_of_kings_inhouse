## 二、数据模型设计

### 2.1 枚举类型

```python
class Role(Enum):
    TOP = "上单"      # 🛡️
    JUNGLE = "打野"   # ⚔️
    MID = "中单"      # 🧙‍♂️
    ADC = "射手"      # 🏹
    SUPPORT = "辅助"  # 🚀

class SignupMode(Enum):
    RANDOM = "random"           # 随机分组模式
    PREFERENCE = "preference"   # 智能分组模式

class TeamSide(Enum):
    RED = "红方"
    BLUE = "蓝方"

class MatchStatus(Enum):
    PENDING_ARCHIVE = "待归档"
    ARCHIVED = "已归档"
    DISCARDED = "已废弃"
```

### 2.2 核心实体

**Player（玩家）**
```python
@dataclass
class Player:
    user_id: str                    # 唯一标识
    nickname: str                   # 昵称
    chat_id: str                    # 群组ID
    signup_mode: SignupMode         # 报名模式
    preferred_roles: list[Role]     # 偏好分路（0或2个）
    signup_ts: int                  # 报名时间戳
    expires_at_ts: int              # 过期时间戳
```

**SignupPool（报名池）**
```python
@dataclass
class SignupPool:
    chat_id: str                    # 群组ID
    players: dict[str, Player]      # user_id -> Player
    queue_order: list[str]          # 报名顺序
    updated_ts: int                 # 更新时间戳
```

**TeamAssignment（队伍分配）**
```python
@dataclass
class TeamAssignment:
    side: TeamSide                  # 红方/蓝方
    role_to_user: dict[Role, str]   # 位置 -> user_id
    score: int                      # 偏好命中分
```

**MatchRecord（对局记录）**
```python
@dataclass
class MatchRecord:
    match_id: str                   # 对局ID
    chat_id: str                    # 群组ID
    created_ts: int                 # 创建时间戳
    algorithm: str                  # 算法类型（random/greedy/mcmf）
    red_team: TeamAssignment        # 红方阵容
    blue_team: TeamAssignment       # 蓝方阵容
    participants: list[str]         # 参与者列表
    winner: str | None              # 胜者（RED/BLUE/DRAW）
    status: MatchStatus             # 状态
    preference_score_total: int     # 总偏好分
    meta: dict[str, str]            # 元数据
```

**ChatState（群组状态）**
```python
@dataclass
class ChatState:
    chat_id: str                    # 群组ID
    signup_pool: SignupPool         # 报名池
    history: list[MatchRecord]      # 历史记录
    pending_archive_match_id: str | None  # 待归档对局ID
    config: ChatConfig              # 配置
    last_midnight_reset_date: str   # 最后午夜重置日期
```

**GlobalState（全局状态）**
```python
@dataclass
class GlobalState:
    version: int                    # 版本号
    generated_at_ts: int            # 生成时间戳
    chats: dict[str, ChatState]     # chat_id -> ChatState
```

## 四、核心算法设计

### 4.1 最小费用最大流（MCMF）

**建图方式**：
- 源点 S → 10个玩家节点 P_i（容量1，费用0）
- 玩家节点 P_i → 10个槽位节点 Slot(team, role)（容量1，费用 cost(i,j)）
- 槽位节点 Slot_j → 汇点 T（容量1，费用0）

**费用函数**（越小越优）：
- 命中第一偏好：0
- 命中第二偏好：2
- 随机报名玩家：4
- 非偏好分配：7
- 随机扰动：+ random(0..2)

**二阶段优化**：
1. MCMF 得到全局可行解
2. 1-2步换位优化（缩小红蓝偏好差、降低非偏好集中度）

### 4.2 贪心+回溯算法

**流程**：
1. 锁定唯一位置的用户（强约束）
2. 对多选用户进行回溯搜索
3. 计算满意度总分（首选10分，次选5分）
4. 随机打破平局

## 六、关键技术实现

### 6.1 并发控制

```python
# 全局状态锁
state_lock = asyncio.Lock()

# 群组级锁
chat_locks: dict[str, asyncio.Lock] = {}

async def with_chat_lock(chat_id: str):
    if chat_id not in chat_locks:
        chat_locks[chat_id] = asyncio.Lock()
    return chat_locks[chat_id]
```

### 6.2 数据持久化（原子写）

```python
def save_state(state: GlobalState):
    tmp_path = "data/hok_inhouse_state.json.tmp"
    target_path = "data/hok_inhouse_state.json"
    bak_path = "data/hok_inhouse_state.json.bak"

    # 1. 写入临时文件
    with open(tmp_path, 'w', encoding='utf-8') as f:
        json.dump(state.to_dict(), f, ensure_ascii=False, indent=2)
        f.flush()
        os.fsync(f.fileno())

    # 2. 备份旧文件
    if os.path.exists(target_path):
        shutil.copy2(target_path, bak_path)

    # 3. 原子替换
    os.replace(tmp_path, target_path)
```

### 6.3 过期机制

```python
# TTL: 5小时（可配置4-6小时）
DEFAULT_TTL = 5 * 3600

# 懒清理：每次命令入口执行
async def cleanup_expired(chat_id: str):
    now = int(time.time())
    pool = get_signup_pool(chat_id)
    expired_users = [
        uid for uid, player in pool.players.items()
        if player.expires_at_ts < now
    ]
    for uid in expired_users:
        del pool.players[uid]
        pool.queue_order.remove(uid)

# 次日凌晨清空
async def check_midnight_reset(chat_id: str):
    state = get_chat_state(chat_id)
    today = datetime.now().strftime("%Y-%m-%d")
    if state.last_midnight_reset_date != today:
        state.signup_pool.players.clear()
        state.signup_pool.queue_order.clear()
        state.last_midnight_reset_date = today
```

## 八、风险与挑战

### 8.1 技术风险

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| MCMF 算法复杂度高 | 性能问题 | 提供贪心算法作为备选 |
| 并发竞态条件 | 数据不一致 | 细粒度锁 + 原子写 |
| 状态文件损坏 | 数据丢失 | 备份恢复机制 |
| 跨平台 Emoji 兼容性 | 显示异常 | 降级策略（文本备注） |

### 8.2 业务风险

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 位置冲突无解 | 无法分组 | 提示用户调整 + 随机分组兜底 |
| 管理员权限滥用 | 数据被清空 | 操作日志 + 恢复机制 |
| 数据过期策略不当 | 用户体验差 | 可配置 TTL + 明确提示 |
